# Technology Stack

**Analysis Date:** 2026-05-05

## Languages

**Primary:**
- Rust 2021 edition - Frontend web application (WASM compilation target)
- CSS 3 - Styling via Tailwind CSS
- HTML 5 - Templates via Dioxus RSX

**Secondary:**
- JavaScript (minimal) - Browser API bindings via wasm-bindgen, web-sys

## Runtime

**Environment:**
- WebAssembly (wasm32-unknown-unknown target)
- Browser environment with localStorage and DOM APIs

**Package Manager:**
- Cargo (Rust) - Primary dependency management
- npm (Node.js) - Tailwind CSS utilities only

## Frameworks

**Core:**
- Dioxus 0.6.1 - React-like framework for Rust web UI (`src/app.rs`)
  - Features: web platform, router support
- Tailwind CSS - Utility-first CSS framework (run via `npx tailwindcss` CLI)

**Testing:**
- wasm-bindgen-test 0.3 - Browser-based WASM unit test runner
- mockito 1.2 - HTTP mocking for integration tests

**Build/Dev:**
- Dioxus CLI (dx) - Development server, hot reload, WASM bundling
- tailwindcss (npm) - CSS utility generation from `input.css`

## Key Dependencies

**HTTP & Networking:**
- reqwest 0.12.15 (features: json) - Async HTTP client for REST API calls (`src/api.rs`)
  - Used for all backend communication (fetch_auth_info, get_sales_persons, post bookings, etc.)

**WASM Bindings:**
- wasm-bindgen 0.2.97 - Rust ↔ JavaScript FFI for DOM access
- wasm-bindgen-futures 0.4.47 - Async/await support in WASM
- web-sys 0.3.77 - Browser APIs (Window, Navigator, Document, Storage, MediaQueryList, etc.)
  - Features include: Window, Document, Element, HtmlTextAreaElement, CssStyleDeclaration, Storage, KeyboardEvent, MouseEvent
- js-sys 0.3.77 - Low-level JavaScript bindings

**Serialization & Type Safety:**
- serde 1.0.219 (features: rc) - JSON serialization/deserialization
- serde_json 1.0.140 - JSON parsing
- rest-types (local workspace dependency, path: `rest-types/`) - Shared type definitions with backend
  - Includes: ShiftplanTO, SalesPersonTO, BookingTO, ExtraHoursTO, etc.

**Async Runtime & Utilities:**
- futures 0.3.30 - Future combinators and utilities
- futures-util 0.3.30 - Extended future utilities
- async-recursion 1.1.1 - Async recursion support
- gloo-timers 0.3.0 (features: futures) - Browser timer integration (setTimeout, setInterval)
- thiserror 2.0.17 - Error type derivation (`src/error.rs`)

**Data & Time:**
- uuid 1.17 (features: v4, js) - UUID generation and parsing (used for entity IDs)
- time 0.3.41 - Date/time handling with serde support
  - Features: macros, serde-human-readable, parsing, formatting
- manganis 0.6.2 - Asset management for bundling

**Logging:**
- tracing 0.1.41 - Structured logging (imported but minimal usage in frontend)
- dioxus-logger 0.6.2 - Dioxus-integrated logging

## Configuration

**Environment:**
- Loaded from `assets/config.json` at runtime via `load_config()` in `src/api.rs`
- Config structure defined in `src/state/config.rs`:
  - `backend`: Backend URL (proxied via Dioxus.toml in development)
  - `application_title`: Display title (default: "Shifty")
  - `is_prod`: Production flag
  - `env_short_description`: Environment label (default: "DEV")
  - `show_vacation`: Feature flag for vacation functionality

**Build:**
- `Dioxus.toml` - Main development and build configuration
  - App name: shifty-dioxus
  - Default platform: web
  - Output directory: dist
  - Asset directory: assets
  - Style: /tailwind.css (generated by Tailwind CLI)
  - Backend proxy configuration for 23 API endpoints (http://localhost:3000/*)

**CSS/Styling:**
- `tailwind.config.js` - Tailwind customization
  - Content scanning: ./src/**/*.{rs,html,css}, ./dist/**/*.html
  - Custom theme extensions: design token colors (bg, surface, accent, etc.)
  - Typography scale: micro, small, body, lg, h2, h1, display
  - Font families: Inter (sans), JetBrains Mono (mono)
  - Safelist: state-dependent classes (bg-bad-soft, text-warn, border-accent, etc.)

- `input.css` - Base Tailwind directives
  - Design token CSS variables for light and dark themes
  - Custom utilities: scale-down-50, scale-down-75, scale-down-100 (zoom-based)
  - Modal animations: shifty-modal-fade, shifty-modal-pop, shifty-modal-slide-*
  - Form input focus styles

**Localization:**
- i18n module in `src/i18n/` with support for English, German, and Czech
  - Enum-based keys defined in `mod.rs`
  - Locale translations in: en.rs, de.rs, cs.rs
  - Locale enum: En, De, Cs with parsing from string
  - Date formatting per locale

## Platform Requirements

**Development:**
- Rust toolchain (2021 edition)
- wasm32-unknown-unknown target installed (`rustup target add wasm32-unknown-unknown`)
- Node.js and npm (for Tailwind CSS CLI)
- Dioxus CLI (`cargo install dioxus-cli`)

**Build:**
```bash
# Terminal 1: CSS compilation
npx tailwindcss -i ./input.css -o ./assets/tailwind.css --watch

# Terminal 2: Development server
dx serve --hot-reload  # Compiles Rust → WASM, serves on http://localhost:8080

# Production build
dx build  # Outputs optimized WASM and assets to dist/
```

**Production:**
- Web server serving static WASM bundle and assets from `dist/` directory
- Backend server at configurable URL (via assets/config.json)
- Browser with WebAssembly support (modern browsers: Chrome, Firefox, Safari, Edge)
- localStorage support for UI preferences (working hours layout)

**API Proxy (Development only):**
- Dioxus dev server proxies 23 backend endpoints from http://localhost:3000 to WASM app
- Production requires CORS headers or proxy configuration at deployment
